# About the project
This project aims to add new AI based features to [Monika After Story mod](https://github.com/Monika-After-Story/MonikaModDev) with the submod API.
It's using multiple AI models:
- [text-generation-webui](https://github.com/oobabooga/text-generation-webui) or [SillyTavern](https://github.com/SillyTavern/SillyTavern) as chat backends
- [TTS Coqui-AI](https://github.com/coqui-ai/TTS), [XTTS](https://huggingface.co/coqui/XTTS-v2), and [Tortoise-TTS](https://github.com/152334H/tortoise-tts-fast) for Text to Speech
- [OpenAI Whisper](https://github.com/openai/whisper) with [microphone option](https://github.com/mallorbc/whisper_mic) for Speech to Text
- [Emotion detection from text model](https://huggingface.co/michellejieli/emotion_text_classifier) is also used linked with the chatbot
- NLI Classification with [deberta](https://huggingface.co/sileod/deberta-v3-base-tasksource-nli)
- [sentence-transformers/all-mpnet-base-v2](https://huggingface.co/sentence-transformers/all-mpnet-base-v2) for Long Term Memory semantic search

Check the discord [server](https://discord.gg/2RsPuaDxEn) if you have some questions or if you want to be up to date with new fixes and releases !

# Getting Started

## Prerequisites

The submod is compatible with the latest version of MAS (v0.12.15). It has been tested on Windows 10/11 with Python 3.9. A manual installation is also possible on Linux or MacOS.

## Installation

### :eight_spoked_asterisk: User version installation (Windows)

This version includes conversing with chatbots or full voicing of the game, the two modes might not be compatible so it is preferable to use them separetely.

To install the user version with executables, a tutorial is available [here](https://github.com/Rubiksman78/MonikA.I/wiki/Installation-tutorial-(after-2.0)) after version 2.0.

### :penguin: Python version (Linux or MacOS)

Wiki page [HERE](https://github.com/Rubiksman78/MonikA.I/wiki/Installation-Tutorial-(Linux,MacOS))

### Additional dependencies for Long Term Memory

If you want to use the Long Term Memory feature, install the additional dependencies:

```bash
pip install scikit-learn sentence-transformers zarr
```

The `sentence-transformers/all-mpnet-base-v2` model (approximately 420MB) will be downloaded automatically on first launch when Long Term Memory is enabled.

# Usage

## :fire: Features

- Speak without scripted text with Monika using any chatbots compatible with `text-generation-webui` or `SillyTavern`
- Hear Monika speak with a Text to Speech module using extracts of voiced dialogues
- Talk with your own voice with Monika thanks to Speech recognition
- Let Monika control actions in the game directly from your chat with her
- Long Term Memory allows Monika to remember past conversations across sessions

## :star2: Chatbots

Be sure to follow the instructions [here](../../wiki/Install-Local-Chatbots-locally) to use conversational AI models running locally.

## :brain: Long Term Memory

The Long Term Memory (LTM) system allows Monika to remember past conversations and recall them in future sessions. It uses semantic search to find relevant memories based on what you are currently talking about, rather than simple keyword matching.

### How it works

- When you chat with Monika, your messages and her responses are stored in a local database alongside their semantic embeddings (768-dimensional vectors generated by the `sentence-transformers/all-mpnet-base-v2` model).
- Text is stored in a SQLite database. Embedding vectors are stored in Zarr format on disk.
- On each new message, the system searches for semantically similar past conversations using cosine distance and injects the most relevant memories into Monika's context before she responds.
- **Memories saved during the current session are NOT used for injection until the next session.** This prevents the current conversation from being flooded back into Monika's context. You can override this with the "Force reload memories" button.
- Messages shorter than 100 characters (configurable) are not saved, to avoid storing trivial exchanges.
- Duplicate messages are automatically detected and skipped.

### Enabling Long Term Memory

1. **Login screen**: Set "Enable Long Term Memory" to **Yes** in the startup configuration screen. This controls whether the LTM system initializes when the server starts.
2. **In-game settings**: Open the submod settings panel (Settings > Submods > AI_submod). The Long Term Memory section contains the following controls:

| Setting | Description |
|---------|-------------|
| **Enable memory injection** | Toggle whether relevant past memories are injected into Monika's context before each response. |
| **Enable memory saving** | Toggle whether new conversations are stored to the memory database. |
| **Force reload memories** | Reload memory embeddings from disk into RAM. This makes memories saved during the current session available for injection immediately. |
| **View memory statistics** | Display the number of memories currently loaded in RAM, saved to disk, and how many were used in the last injection. |
| **Destroy all memories** | Permanently delete all stored memories. Monika will ask for confirmation before proceeding. |

Memory injection and memory saving can be toggled independently. For example, you can disable saving while keeping injection on to use existing memories without adding new ones.

### Configuration

The LTM system is configured through `ltm_config.json` in the project root:

```json
{
    "ltm_context": {
        "injection_location": "BEFORE_NORMAL_CONTEXT",
        "memory_context_template": "Monika's memory log from past conversations with {name1}: ...",
        "memory_template": "{time_difference}, {memory_name} said: \"{memory_message}\""
    },
    "ltm_writes": {
        "min_message_length": 100
    },
    "ltm_reads": {
        "max_cosine_distance": 0.60,
        "num_memories_to_fetch": 2,
        "memory_length_cutoff_in_chars": 1000
    }
}
```

| Parameter | Default | Description |
|-----------|---------|-------------|
| `injection_location` | `BEFORE_NORMAL_CONTEXT` | Where to inject memories relative to the AI's context. Also supports `AFTER_NORMAL_CONTEXT_BUT_BEFORE_MESSAGES`. |
| `memory_context_template` | *(see file)* | Template wrapping all retrieved memories. Uses `{name1}` (player) and `{all_memories}` placeholders. |
| `memory_template` | *(see file)* | Template for each individual memory. Uses `{time_difference}`, `{memory_name}`, and `{memory_message}` placeholders. |
| `min_message_length` | `100` | Minimum character length for a message to be saved to memory. |
| `max_cosine_distance` | `0.60` | Maximum cosine distance for a memory to be considered relevant (lower = stricter matching). |
| `num_memories_to_fetch` | `2` | Number of memories to retrieve per query. |
| `memory_length_cutoff_in_chars` | `1000` | Maximum character length of a single memory when injected into context. |

### Memory storage

Memories are stored per-character in `user_data/bot_memories/`. Each character gets its own subdirectory containing:
- `long_term_memory.db` - SQLite database with message text, speaker name, and timestamps
- `long_term_memory_embeddings.zarr` - Zarr array storing the 768-dimensional embedding vectors

If a legacy (non-namespaced) database is detected in the `bot_memories/` directory, it will be used as-is with a warning that character namespacing is disabled.

### Credits

The Long Term Memory system is based on [a4memignir](https://github.com/Delcos/a4memignir) by Delcos, adapted for the MonikA.I. submod architecture.
